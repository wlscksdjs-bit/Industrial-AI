# 머신러닝 강의 요약 - 2026년 4월 29일

본 강의에서는 대표적인 화이트박스(White Box) 모델인 **의사결정 트리(Decision Tree)**의 기본 구조와 동작 원리, 노드 분할을 위한 불순도 지표(엔트로피, 정보 이득, 지니 지수), 트리 모델의 한계 극복을 위한 C4.5 및 CART 알고리즘, 연속형 변수의 처리 및 **회귀 트리(Regression Tree)** 설계 방식에 대하여 학습했습니다.

---

## 1. 의사결정 트리(Decision Tree)의 개요 및 트리 구조

의사결정 트리는 질문을 거듭해가며 데이터를 분류하거나 예측해나가는 알고리즘으로, 스무고개(20 Questions) 게임과 유사한 방식으로 동작합니다.

*   **화이트박스(White Box) 모델**: 딥러닝 등의 블랙박스 모델과 달리 최종 판단에 도달한 의사결정 경로를 명확하게 추적하고 설명할 수 있어 해석력이 매우 높습니다.
*   **노드 구성 요소**:
    *   **뿌리 노드 (Root Node)**: 트리 구조가 시작되는 최상단 노드입니다.
    *   **자식 노드 / 중간 노드 (Internal Node)**: 조건 분기가 일어나는 노드입니다.
    *   **끝마디 노드 (Leaf Node / Terminal Node)**: 최종 클래스 분류값 또는 회귀 예측치가 결정되어 더 이상 분기하지 않는 마지막 노드입니다.
    *   **트리 깊이 (Tree Depth)**: 뿌리 노드부터 끝마디 노드까지 이르는 최대 경로 길이로, 과대적합을 막는 주요 하이퍼파라미터입니다.

---

## 2. 노드 분할 기준: 엔트로피 vs. 지니 지수

노드를 자식 노드로 분할할 때는 분할 후 데이터의 **불확실성(불순도, Impurity)**이 가장 낮아지는 방향을 선택해야 합니다.

### 1) 엔트로피 (Entropy)와 정보 이득 (Information Gain)
*   **엔트로피**: 데이터셋의 무질서도 또는 불확실성을 나타냅니다. $0$에 가까울수록 순수도가 높고(모두 한 클래스), $1$에 가까울수록 불확실성이 높습니다. (확률이 항상 $1$ 이하이므로 로그 연산 후 양수화를 위해 앞에 음수 기호 $-$를 붙입니다.)
    
    $$H(D) = -\sum_{i=1}^C p_i \log_2(p_i)$$

*   **정보 이득 (Information Gain)**: 부모 노드의 엔트로피에서 분할 후 자식 노드 엔트로피의 가중 평균을 뺀 값입니다. **ID3 알고리즘**은 정보 이득을 최대화하는 변수를 분기 기준으로 선택합니다.
    
    $$\text{Gain}(D, A) = H(D) - H(D \mid A)$$

### 2) C4.5 알고리즘과 정보 이득비 (Gain Ratio)
*   **정보 이득의 한계**: 고유한 카테고리가 너무 많은 속성(예: 고객 ID, 주민번호 등)을 기준으로 노드를 쪼개면 개별 엔트로피는 $0$이 되지만, 이는 단순히 암기한 오버핏 모델이 되어 새로운 데이터 예측 능력이 소실됩니다.
*   **보완책 (C4.5)**: 분할 정보량(Split Info)으로 정보 이득을 나누어 페널티를 부과하는 **정보 이득비(Gain Ratio)**를 기준으로 사용하여 다중 분할 속성의 오버핏을 억제합니다.

### 3) 지니 지수 (Gini Index)와 CART 알고리즘
*   **지니 지수**: 데이터셋에서 임의로 선택한 데이터가 잘못 분류될 확률을 측정합니다. 엔트로피와 달리 복잡한 **로그(log) 연산이 없어 컴퓨터 연산 속도가 매우 빠릅니다**.
    
    $$\text{Gini}(D) = 1 - \sum_{i=1}^C p_i^2$$

*   **CART 알고리즘**: 지니 지수를 최소화하는 방향으로 분할하며, 항상 자식 노드를 정확히 2개로만 쪼개는 **이진 분할(Binary Split)**만 수행합니다.

## 3. 연속형 변수의 분할 처리 기법

의사결정 트리에서 수치형 연속형 변수를 분기 기준으로 삼을 때 사용하는 메커니즘입니다.

1.  **정렬**: 해당 연속형 변수의 값들을 오름차순으로 1차 정렬합니다.
2.  **경계 후보 탐색**: 정렬된 상태에서 클래스 라벨이 바뀌는 지점(경계)들을 탐색합니다.
3.  **임계값(Threshold) 계산**: 인접한 두 값의 평균값을 경계 임계값 후보로 설정합니다.
4.  **엔트로피 계산**: 각 후보 임계값을 기준으로 "이 값보다 큰가?"를 묻는 이진 조건의 엔트로피를 계산하여 가장 우수한 경계값을 확정합니다.

---

## 4. 회귀 트리 (Regression Tree)의 동작 원리

종속 변수 $y$가 연속적인 수치일 때는 엔트로피나 지니 지수를 사용할 수 없습니다.

*   **분산 감소량 (Variance Reduction)**: 자식 노드들의 수치값 분산(Variance)이 최소가 되도록 쪼갭니다.
*   **예측값 반환**: 최종 리프 노드에 도달한 학습 데이터 수치값들의 **평균(Mean)**을 최종 예측치로 반환합니다.
*   **변수 재사용성**: 일반 범주형 변수와 달리, 연속형 변수는 임계값을 달리하여 다른 레벨(깊이)의 노드에서 **여러 번 재사용**하여 정교한 분기를 만들 수 있습니다.

---

## 5. 트리 규제: 가지치기 (Pruning)

훈련 데이터에 너무 완벽하게 맞춰진 깊은 트리는 오버핏을 유발하므로 조절이 필수적입니다.
*   **사전 가지치기 (Pre-pruning)**: 트리 깊이의 최대 한도(max_depth)나 노드 최소 샘플 수 등을 학습 전 미리 규정하여 특정 조건 도달 시 분할을 정지합니다.
*   **사후 가지치기 (Post-pruning)**: 트리를 끝까지 다 생성한 후, 검증 데이터셋을 기준으로 검증 성능을 떨어뜨리는 하위 노드들을 다시 잘라내어 리프 노드로 단순화합니다.
